# 고속도로 VDS 데이터 수집기

data.ex.co.kr 의 VDS 데이터를 **데이터 소스별·기간별로 선택해 CSV로 저장**하는 웹 앱입니다.
백엔드(FastAPI)가 순수 HTTP로 파일을 받아 gzip 해제 후 CSV로 저장하고,
React 프론트엔드에서 기간 선택·진행률·저장 현황을 확인합니다. (브라우저/Selenium 불필요)

## 주요 기능

- **데이터 소스별 기간 선택** — 소스(84/87/F5)마다 시작일·종료일을 따로 지정
- **신청자 정보(받는 주체)** — 소속/지역/성별/나이/활용목적을 입력해 '받는 주체'로 서버에 기록
- **실시간 진행률** — 전체/소스별 프로그래스 바와 현재 처리 날짜 표시
- **저장 현황** — 소스별로 통합본에 담긴 '일수'를 별도 영역에 표시
- **다운로드 즉시 단일화** — 받는 즉시 소스별 **하나의 통합 CSV**에 이어붙여 저장 (일자별 파일 없음, 헤더 1회)
- **재개 지원** — 이미 받은 날짜·데이터 없는 날짜는 자동으로 건너뜀
- **취소** — 진행 중인 작업을 즉시 중단
- **순수 HTTP 다운로드** — 브라우저/드라이버 없이 서버가 직접 파일 수집

## 구조

```
ex_collector/
├─ backend/                FastAPI 서버 (순수 HTTP 다운로드)
│  ├─ config.py            소스 정의(84/87/F5)·경로·네트워크 설정
│  ├─ ex_client.py         data.ex.co.kr 클라이언트 (검색→다운로드→gzip해제)
│  ├─ storage.py           CSV 저장·개수 집계·재개(중복 건너뛰기)
│  ├─ jobs.py              백그라운드 다운로드 작업 + 진행률 추적
│  ├─ app.py               API 라우트 + 프론트엔드 서빙  ← 진입점
│  ├─ requirements.txt
│  └─ data/                소스별 CSV 저장 위치 (실행 시 생성)
└─ frontend/               React + Vite 대시보드
   ├─ src/                 App / api / components(SourceCard·StatusPanel·ProgressBar)
   └─ dist/                빌드 결과 (백엔드가 이 폴더를 서빙)
```

## 실행 방법

**필요:** Python 3, Node.js(npm) · 인터넷 연결(data.ex.co.kr 접근)

> `frontend/dist/` 와 `backend/.venv/` 는 `.gitignore` 대상이라 **GitHub에는 포함되지 않습니다.**  
> 저장소를 새로 clone 한 뒤에는 아래 **초기 설정(1회)** 을 거친 다음 서버를 실행하세요.

### 초기 설정 (최초 1회)

```bash
# 저장소 받기
git clone https://github.com/kcleverp/data-collector.git
cd data-collector

# 백엔드
cd backend
python -m venv .venv
.venv\Scripts\activate            # PowerShell: .venv\Scripts\Activate.ps1
pip install -r requirements.txt

# 프론트 빌드 (8000 포트 UI에 필수)
cd ..\frontend
npm install
npm run build
```

### 서버 실행

```bash
cd backend
.venv\Scripts\activate
python app.py
```

브라우저에서 **http://127.0.0.1:8000** 접속.

- 화면에 **「신청자 정보 (받는 주체)」** 5개 선택란이 보이면 정상입니다.
- UI가 예전 버전처럼 보이면 **Ctrl+Shift+R**(강력 새로고침) 하세요.
- 프론트 소스를 수정했다면 `npm run build` 후 서버를 재시작하세요.

### 프론트엔드 개발 모드 (선택)

UI를 수정·확인할 때는 빌드 없이 dev 서버를 쓸 수 있습니다. **백엔드(8000)도 함께 켜야** 합니다.

```bash
# 터미널 1 — backend
cd backend
.venv\Scripts\activate
python app.py

# 터미널 2 — frontend
cd frontend
npm install          # 최초 1회
npm run dev          # http://localhost:5173 (/api 는 8000으로 프록시)
```

배포(8000) 반영: `npm run build` → `backend/app.py`가 새 `frontend/dist/`를 서빙.

## 사용 흐름

1. **소스·기간 선택** — 받을 소스(체크박스)를 켜고 시작일·종료일을 고릅니다. (가능 기간 안내됨)
2. **신청자 정보 입력** — 소속/지역/성별/나이/활용목적을 선택합니다. '받는 주체'로 서버에 기록되는 필수 정보입니다.
3. **수집 시작** — 진행률 바가 채워지고, 받는 즉시 **소스별 하나의 통합 CSV**에 이어붙여 저장됩니다.

- 저장 위치(통합본): `backend/data/<소스이름>/<소스번호>_통합.csv`
  - 폴더는 **소스 이름**으로 생성됩니다. 예) `data/VDS_정체길이/87_통합.csv`
  - 진행 기록 `.progress.json`이 담긴 날짜를 추적합니다(재개용).
- `data/` 폴더는 **실행 시 자동 생성**되며 `.gitignore`로 저장소에서는 제외됩니다.
  (예전 `84/87/F5` 폴더나 일자별 파일이 있으면 서버 시작 시 소스 이름 폴더의 **통합본으로 자동 합쳐집니다**.)
- 이미 받은 날짜·데이터 없는 날짜는 자동으로 **건너뜁니다**(재개 지원).

## 데이터 구조 및 입도

모든 소스는 **파일 1개 = 하루치**로 저장되지만, 내부 시간 입도와 단위는 소스마다 다릅니다.
(2024-01-01 실측 기준)

| 소스 | 이름 | 시간 입도 | 단위 | 하루 행수(약) | 주요 측정값 |
|---|---|---|---|---|---|
| **84** | VDS 교통량·속도·지정체 | **15분** (00:00~23:45, 96개) | 지점(VDS_ID) 7,235개 | 69.5만 | 교통량, 평균속도 |
| **87** | VDS 정체길이 | **15분** (96개) | 지점(VDS_ID) × 차로·방향 | 100만 | VDS존 정체길이(콘존별) |
| **F5** | 구간별 소통일관성지수(TCI) | **1시간** (00~23, 24개) | 구간(콘존) 1,267개 | 3만 | TCI1지표 |

- **84 · 87** : 한 지점당 하루 96개(24시간 × 4) → **15분 간격**.
- **F5** : 한 구간당 하루 24개 → **1시간 간격**. (다른 두 소스와 입도가 다름에 주의)
- 시간 컬럼: 84·87 은 `기준시`(HHMM), F5 는 `집계시`(HH).

### 컬럼 구성

- **84** — `기준시간, 기준시, 기준일, VDS_ID, 요일명, 지점이정, 노드명, 도로이정, 노선번호, 도로명, 교통량, 평균속도`
- **87** — `기준일, 요일명, 기준시, 도로이정, 노선번호, 도로명, 기점종점방향구분코드, 차로유형구분코드, VDS_ID, 지점이정, 콘존ID, 콘존명, VDS존길이1~4`
- **F5** — `집계시, 도로명, 노선구성순번, 방향구분, TCI1지표, 콘존명, 콘존길이`

## 주요 API

| 메서드 | 경로 | 설명 |
|---|---|---|
| GET | `/api/sources` | 소스 목록 + 가능 날짜 범위 + 담긴 일수 + 통합본 정보 |
| GET | `/api/survey-options` | 신청자 설문 항목/선택지 |
| GET | `/api/info` | 저장 경로(dataDir) 등 |
| POST | `/api/download` | 다운로드 시작 `{selections, survey}` → `{jobId}` |
| GET | `/api/jobs/{id}` | 진행률/받는 주체 조회 |
| POST | `/api/jobs/{id}/cancel` | 작업 취소 |

## 참고

- 로그인 불필요. 다운로드 전 신청자 설문 5개(소속/지역/성별/나이/활용목적)는 **필수**이며, 프론트에서 입력한 값이 '받는 주체'로 서버에 기록됩니다. (데이터 내용 자체에는 영향 없음)
- 소스 3종: `84`(교통량·속도·지정체), `87`(정체길이), `F5`(구간별 소통일관성지수 TCI).
- CSV 인코딩은 원본 그대로(cp949)입니다. Excel에서 바로 열립니다.
